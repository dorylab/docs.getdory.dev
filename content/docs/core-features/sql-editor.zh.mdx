# SQL Editor

SQL Editor 是 Dory 的核心工作区，也是大多数数据分析任务开始的地方。它将 SQL 编写、执行结果、Schema 上下文、图表分析与 AI 能力整合在同一工作流中，帮助你更快完成从问题到答案的过程。

## 为什么使用 Dory SQL Editor

传统数据库工具通常将编辑器、表结构浏览、结果查看和 AI 工具分散在不同位置。Dory 将这些能力整合在一起，让分析过程更流畅。

你可以获得：

- 更准确的 SQL 自动补全与字段提示。
- 更快的查询编写与多轮迭代体验。
- 查询结果与图表无缝切换。
- 基于当前上下文的 AI SQL 辅助。
- 查询保存与复用能力。

## 智能自动补全

连接数据库后，Dory 会读取 Schema 信息，提供更准确的提示能力，包括：

- 表名补全
- 字段名补全
- Schema / Database 提示
- SQL 关键字建议

这可以显著减少拼写错误与来回查表成本。

## 多标签页工作流

同时处理多个任务时，你可以使用多个查询标签页，例如：

- 一个标签页查看原始数据
- 一个标签页编写分析 SQL
- 一个标签页调试查询性能
- 一个标签页生成图表结果

## 快速执行与结果查看

运行查询后，你可以直接查看：

- 表格结果
- 行数与返回状态
- 查询耗时
- 图表视图（适用时）

## 与 AI 协作

SQL Editor 与 AI 能力深度结合。你可以：

- 生成 SQL 草稿
- 优化已有 SQL
n- 解释复杂查询逻辑
- 将自然语言转换为查询语句
- 基于当前结果继续分析

## 保存常用查询

将高频 SQL 保存为可复用查询，减少重复劳动，并沉淀团队知识。

## 推荐使用流程

1. 在 Explorer 中找到目标表。
2. 打开 SQL Editor 新建标签页。
3. 使用自动补全快速编写 SQL。
4. 执行查询并查看结果。
5. 使用 AI 继续优化或生成下一步分析。
6. 保存有价值的查询。

## 示例查询

```sql
select city, count(*) as users
from users
group by city
order by users desc
limit 10;
```

运行后可直接切换图表视图查看城市分布。

## 最佳实践

- 先使用 `limit` 验证数据结构，再扩大查询范围。
- 对复杂 SQL 分步骤编写，逐步验证结果。
- 常用分析语句建议保存复用。
- 使用 AI 生成初稿，再人工审核和调整。
- 在生产环境优先使用只读连接。
