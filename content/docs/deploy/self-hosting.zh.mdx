# 自托管

公开仓库当前主要提供了基于 Docker 的自托管路径。

## 自托管可以带来什么

- 由你自己控制部署环境与凭据。
- 在本地或团队环境中运行可通过浏览器访问的 Dory 工作区。
- 明确配置产品所使用的 AI 提供商与模型。

## 关键环境变量

| 变量 | 作用 |
| --- | --- |
| `DS_SECRET_KEY` | 应用运行时使用的密钥。 |
| `BETTER_AUTH_SECRET` | 认证流程所需密钥。 |
| `BETTER_AUTH_URL` | 当前部署对外访问的基础地址。 |
| `DORY_AI_PROVIDER` | AI 提供商标识，例如 `openai` 或 `anthropic`。 |
| `DORY_AI_MODEL` | 当前提供商对应的模型名称。 |
| `DORY_AI_API_KEY` | 提供商 API Key。 |
| `DORY_AI_URL` | 提供商或兼容端点所需的基础 URL。 |

## 运维注意事项

- 将 AI 凭据视为部署密钥。
- 仔细匹配 provider、model 与 URL，否则运行时可能直接失败。
- 不要直接把本地快速体验配置照搬到生产环境认证配置中。

## 当前状态

仓库已经让 Docker 部署足够直接，但尚未形成完整的备份、高可用或企业级加固指南。在更大范围部署前，建议自行补齐这些验证。
