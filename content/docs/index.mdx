# Dory Documentation

Dory is an AI-native data workspace for modern databases. It combines a schema-aware SQL editor, conversational AI, database exploration, saved queries, and charting into one workspace.

## What this site covers

- How to get Dory running through the demo, desktop app, Homebrew, or Docker.
- How to connect supported databases and work with Explorer, SQL Editor, and charts.
- How Dory's AI features are grounded in database schema and current query context.
- What is available today for self-hosting, AI configuration, and release tracking.

## Recommended path

1. Start with [Getting Started](./getting-started).
2. Configure [Connections](./guides/connections).
3. Learn the [SQL Editor](./guides/sql-editor) and [AI Chat](./guides/ai-chat).
4. Review [Self-Hosting](./deploy/self-hosting) before running Dory for a team.

## Current product scope

Based on the public repository, Dory currently documents support for:

- ClickHouse with deep operational integration.
- PostgreSQL, MySQL, MariaDB, and SQLite.
- Pluggable AI providers including OpenAI, Anthropic, Google, Qwen, xAI, and OpenAI-compatible APIs.

For recent product changes, see [Release Notes](./release-notes).
